v2.41
資料結構升級：新增 author (作者)、extractionCode (提取碼)、並允許使用者自訂好記的 trip_id。

初始介面大改版：

分為 「建立旅程」 與 「尋找旅程」 兩大分頁。

尋找旅程：整合了 「本機歷史紀錄」 (自動列出您看過的行程) 與 「雲端搜尋」 
搜尋邏輯升級：輸入關鍵字 -> 先查 ID -> 若無，再查 Author (精確匹配) -> 顯示結果列表 -> 點擊後輸入提取碼。

安全提取機制 (Gatekeeper)：

讀取行程時，若偵測到該行程設有提取碼，會彈出 「🔒 安全鎖定」 視窗，必須輸入正確代碼才能查看內容。

舊版 (v2.3) 無碼行程會自動放行，並在設定中提示補設密碼。

全新側邊欄 (Drawer) 導航：

左上角新增漢堡選單，點擊滑出側邊欄，方便隨時切換行程或查看歷史紀錄。

新增 「變更行程 ID (搬家)」 功能區塊。

輸入新 ID -> 系統檢查 ID 是否重複 -> 若可用 -> 執行「複製資料 -> 寫入新 ID -> 刪除舊 ID」的搬移程序。

搬移成功後，自動更新瀏覽器網址與歷史紀錄。

📋 v2.42 更新日誌
UI 修復 (航班卡片)：

時間顯示優化：將起飛/抵達時間的字體大小調整為 text-xl (原為 text-2xl)，並移除了粗體 font-black 改為 font-bold，防止在電腦版瀏覽器因字元過寬而被裁切。

隱藏預設圖示：透過 CSS 隱藏了瀏覽器原生的時間選擇器圖示 (時鐘 icon)，釋放更多顯示空間，讓版面更整潔。

輸入代號 (如 JX800) 後點擊，會自動開啟 Google 搜尋該航班，方便使用者快速對照並填寫起降時間。

🛠️ v2.43 修正內容
行程時間輸入修復：在行程列表的透明時間輸入框上，補回了點擊觸發 showPicker() 的事件。現在點擊時間區塊，會強制彈出瀏覽器的時間選擇器。

航班時間輸入優化：雖然航班卡片的時間是可見的，但也補上了點擊觸發選擇器的功能，讓操作更直覺（不需要精準點擊數字也能修改）。

📋 v2.5 更新日誌 (Changelog)
介面優化 (Header)：

隱藏日期列：現在只有在「行程 (Plan)」分頁時才會顯示上方的日期切換卡。切換到票券、分帳或翻譯時，日期列會自動隱藏，讓手機螢幕能顯示更多內容。

網頁標題：簡化為「旅遊計畫」。

版本號：移動至側邊選單的最下方。

韓國旅遊神器 (Naver Map 直連)：

當目的地偵測為韓國 (幣別 KRW 或韓文) 時，行程卡片會出現綠色的 N 按鈕。

App 喚醒技術：點擊後會優先嘗試喚醒手機上的 Naver Map App 進行導航 (使用 URL Scheme)。

Fallback 機制：若未安裝 App，系統會在 1.5 秒後自動跳轉至 Naver Map 網頁版 (含防止流量失控的判斷)。

翻譯介面回歸：

恢復 v1 版本的 「三大功能卡片」 設計 (照相翻譯、我說中文、對方說外文)，按鈕更大、更適合旅行中快速點擊。

系統核心升級 (流量與效能)：

智慧寫入鎖 (Smart Save Lock)：導入 JSON 深層比對機制。現在只有當資料 「真的有變動」 時，才會向 Firebase 發送寫入請求。這能徹底解決「已同步」一直閃爍的問題，並大幅節省資料庫讀寫流量。
